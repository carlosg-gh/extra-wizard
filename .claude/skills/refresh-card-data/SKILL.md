# Refresh card data

The static card index lives in `public/data/` and is generated by the build-time pipeline.

## Steps

1. Note current counts/coverage from `public/data/meta.json` (if it exists).
2. Run the pipeline (the only networked script):
   ```bash
   pnpm data:refresh
   ```
   For a quick check without writing files: `pnpm data:coverage`.
3. Compare the new `meta.json` `counts` and `coverage` to the previous values and report
   the delta (monsters added, exact/approximate/unparsed movement).
4. Run `pnpm test` (the golden query test validates the new index) and `pnpm build:app`.
5. The artifacts in `public/data/` are committed to the repo — include them in the commit.

## Notes

- Source URLs and fallbacks are in `src/pipeline/sources/yamlYugi.ts`.
- The pipeline exits non-zero if parse coverage falls below 85% — that signals upstream
  wording changes; use the `add-material-grammar-pattern` skill to fix the new cases.
- Pass `--limit=N` to process only the first N records while iterating.
